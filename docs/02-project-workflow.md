## Kanban Pipeline Stages

The Project Manager visualizes projects through the following linear pipeline:

1. **New Project (Draft):** Project created, client assigned, Drive folder linked. Unassigned.
2. **Design Phase:** Assigned to a Web Designer. Designer sees it in their "Active Projects".
3. **Design Review:** Designer submits Figma URL. Project moves here. PM reviews.
    - *Action:* PM approves -> Moves to Development. PM rejects -> Back to Design.
4. **Development Phase:** PM adds `wp-admin` credentials and SOPs, then assigns a Web Developer.
5. **Internal Review:** Developer completes the WordPress build and submits. PM reviews the live link.
6. **Completed:** Project is archived and appears in the history of all involved users.
